# AXM IO

Insertion Order autofiller for AxMedia campaigns. Load a campaign from Monday, reuse a saved IO, or back-fill from a previously generated IO PDF, review the fields, and download a populated IO PDF styled to match the Grapeseed Media template.

**Live site:** https://<username>.github.io/axm-io/

## How it works

Three ways to load a campaign into the form:

1. **Fetch from Monday** — paste a `🟨Ax Submissions` item ID (submission form is linked there) and pull its fields directly.
2. **Start from a previous IO** — pick from the Supabase-backed history of IOs generated by this tool.
3. **Populate from an old IO** — click the button, drop a previously generated IO PDF, and the form back-fills. Useful for remaking an IO with new dates when it isn't in the history feed.

Then review / edit any field (Blended CPM auto-computes from the selected channels) and click **Generate & download IO.pdf** — a styled landscape letter PDF.

### Back-fill from an old IO — two paths

- **Embedded metadata (lossless).** Every IO this tool generates stores its full field snapshot as JSON in the PDF's Keywords metadata (`AXIO1:{…}`). Re-importing one of those restores every field exactly — no OCR.
- **OCR fallback.** Older IOs (pre-metadata) and PandaDoc-signed copies (signing strips the metadata) are flat images with no text layer, so page 1 is OCR'd with Tesseract.js and parsed against the fixed IO layout. Clean fields (title, budget/CPM, flight dates, landing page, channels, geo) recover reliably; freeform cells and media notes are best-effort — **always review before generating.**

## Privacy

All parsing, OCR, rendering, and PDF generation run in the browser. No submission data ever leaves the user's machine.

## Stack

Single self-contained HTML file (~52 KB). External dependencies loaded from cdnjs at runtime:

- PDF.js (reads old-IO metadata + rasterizes pages for OCR)
- Tesseract.js (OCR back-fill for old/signed IOs, lazy-loaded)
- html2canvas (render → canvas)
- jsPDF (canvas → PDF; also embeds the field snapshot in PDF metadata)
- supabase-js (IO history persistence)

No build step. Edit `index.html` directly; updates deploy on push to `main`.
